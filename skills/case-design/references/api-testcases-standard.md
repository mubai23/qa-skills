# 接口测试用例标准说明

本文件定义**接口测试**场景下的用例设计维度与输出要求。与 SKILL 中的通用策略和内置安全检查配合使用。

---

## 一、必须覆盖的维度

### 1.1 请求与参数

- **正常请求**：必填参数齐全、类型与格式正确、在约定范围内的取值。
- **参数校验**：每个必填参数至少 1 条缺失用例；每个有类型/格式约束的参数至少 1 条格式非法用例；有范围/长度的用边界值策略。
- **可选参数**：覆盖不传、传空、传有效值三种情况。

### 1.2 响应与错误码

- **成功响应**：验证 HTTP 状态码、业务码、响应体结构及关键字段与文档一致。
- **错误码**：文档中定义的每个错误码至少对应 1 条用例，明确触发条件与预期返回。
- **未定义错误**：对明显非法请求，预期为 4xx 或文档约定错误码，不应返回 200。

### 1.3 鉴权与权限

- **鉴权**：覆盖未带鉴权、鉴权无效/过期、鉴权格式错误，预期均为未授权类错误。
- **权限**：若存在角色/租户/资源归属，覆盖越权访问，预期为无权限类错误或空数据。

### 1.4 幂等与并发

- **幂等**：对标明幂等的接口，设计重复提交用例，预期仅生效一次且结果一致。
- **并发与限流**：若文档有并发限制，设计超并发请求用例，预期为限流/排队或约定错误。

---

## 二、输出时的表述要求

- **接口标识**：每条用例明确对应接口路径+方法（如 `POST /api/order`）
- **请求说明**：步骤中写清请求体/参数的关键取值或示例
- **预期结果**：必须包含 HTTP 状态码与响应中断言相关的部分
- **数据要求**：若依赖特定环境数据，在前置条件中说明

---

## 三、BR 追溯要求

若上游有 prd-analysis 输出，每条用例备注中标注关联 BR 编号。安全检查用例标注检查类型（如 `SQL注入检查`）。
