# 自动化候选用例标准说明

本文件定义如何从通用测试用例中筛选与标记适合自动化实现的用例。

---

## 一、适合自动化的特征

- **稳定可重复**：步骤与预期结果稳定，不依赖难以复现的时机或人工判断
- **高执行频率**：核心流程、冒烟、回归中常执行的用例
- **断言明确**：预期结果可被程序化校验（返回码、关键字段、页面元素、数据库状态）
- **接口或可脚本化**：接口类用例天然易自动化；UI 类需元素可定位
- **独立或依赖可构造**：依赖可通过 setup/teardown 恢复

## 二、不适合或低优先的特征

- **强主观或探索性**：如界面美观、交互顺畅
- **一次性或低频**：如兼容性矩阵中少量组合
- **环境或数据难以自动化**：如依赖真实支付、短信
- **变更频繁**：需求或 UI 频繁变更，维护成本高

## 三、输出时的表述要求

- **自动化候选标记**：在备注中标注「自动化候选」或「建议自动化」，可标注高/中/低优先级
- **建议说明**（可选）：简短说明建议实现方式、断言要点、数据与环境要求
- **与通用用例一致**：自动化标记是对执行方式的补充，不替代用例设计质量
