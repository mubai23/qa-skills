# 性能测试用例标准说明

本文件定义**性能测试**场景下的用例设计维度与输出要求。与 SKILL 中的通用策略配合使用。

---

## 一、必须覆盖的维度

### 1.1 指标与基线

- **响应时间**：明确被测接口/场景、目标 RT 指标（P95/P99/平均）、压测时长或请求量
- **吞吐量**：明确目标 QPS 或并发数、持续时间、是否允许少量超时
- **资源与容量**：明确资源约束与预期在约束内达到的负载

### 1.2 场景类型

- **基准/基线**：低负载下验证接口可正常响应，作为后续对比基线
- **稳态负载**：给定正常/峰值负载下持续运行，验证 RT、成功率、错误率在约定范围
- **峰值/尖峰**：短时流量突增，验证系统不崩溃、降级策略生效
- **长时间稳定性**：若要求 7×24 运行，明确运行时长与监控指标

### 1.3 瓶颈与退化

- **瓶颈探测**：阶梯加压，说明需观察的指标（CPU、内存、磁盘 I/O、数据库连接等）
- **降级与限流**：设计触发条件与预期行为

---

## 二、输出时的表述要求

- **场景名称**：能看出场景类型+目标指标，如「登录接口-1000QPS稳态-P99<200ms」
- **负载定义**：写清并发数、QPS、持续时间、是否阶梯加压
- **环境与数据**：前置条件中说明压测环境、数据量级
- **通过标准**：明确判定条件（如 P99 RT≤200ms、成功率≥99.9%）
- **观察要点**：列出需关注的监控项
